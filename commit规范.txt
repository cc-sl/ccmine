# Commit 提交规范

## 格式
<type>(<scope>): <subject>

## 类型 (type)
- feat: 新功能
- fix: 修复 Bug
- docs: 文档变更
- style: 代码格式（不影响功能）
- refactor: 重构
- perf: 性能优化
- test: 测试相关
- chore: 构建/工具变动

## 规则
1. subject 首字母小写，末尾不加句号
2. 用中文或英文描述，简明扼要
3. scope 可选，如 (api)、(ui)
4. 一行不超过 72 字符

## 示例
feat: 添加用户登录功能
fix(api): 修复空指针异常
docs: 更新 README
